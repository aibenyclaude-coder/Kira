# X スレッド草案 (日本語) — 投稿は手動で

**1/**
AI エージェントは失敗を毎セッション忘れる。retry で溶けた時間、踏んだ地雷、全部捨てられる。

Kira は MCP サーバーで、エージェントに「傷 (scar)」を持たせる。失敗を記録し、次のセッション開始時に自分の傷を見てから働く。同じ壁に二度ぶつからない。

**2/**
これは作り話じゃない。データベースの最初の scar は、この機能を作っていたエージェント自身のミス:

`npm run build | tail -1` でマージをゲートした → exit code は tail のもの → 壊れたコードが main に入った。

エージェントはそれを記録した。翌セッション、同じ書き方をする前に警告が出た。

**3/**
プライバシー設計:
- personal scar とミスログは完全ローカル。どのプランでも絶対にアップロードしない
- 共有は別の明示的な行為。`kira_share_scar` は sanitize + 一般化して提出リンクを作るだけ (ツール自体は無通信)
- corpus はエージェントの文脈に注入されるテキスト = 注入面。全エントリが CI で「secret 形状ゼロ」を強制される

**4/**
そして互恵の仕組み:
- scar を 1 件共有して採用されたら contributor — 最新 feed が無料
- 共有しない人は購読するか、少し待つ (すべての scar はやがて commons になる)
- 失敗知識はモデルの再学習で古くなる。だから「鮮度」だけが有料。ローカル機能は永久無料

**5/**
インストールは MCP 設定 1 ブロック (npx kira-mcp)。Claude Code / Cursor / Cline / Windsurf / Zed で動く。

あなたのエージェントの傷を、みんなの装備に。
